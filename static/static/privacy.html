Privacy Policy

I don't really know how to write a proper privacy policy, but this application does store your data on the server!

Unencrypted personal data

This data will be stored unencrypted in the database.

  • Your username
  • Your token count
  • The unique identifiers of source domains you have added to your account
  • The unique resource identifier of sources you are subscribed to
  • Object metadata:
    • Which version of this object is currently visible to you
    • Whether or not you have any encrypted data associated with the object

This data is required for core functionality of the service.

Personal data in logs

Server logs (including e.g. the nginx log) may include your IP address, date and time of accesses, URL queries, and similar data.

Unencrypted shared data

This data may be created by or associated with a user account but is generally accessible to everyone (though it is not indexed).

  • Source domains that you own are publicly accessible if you know their random identifier, and may optionally be shown in a public list.
  • Source data (essentially any data fetched by a source domain) may contain any information, including personal information. No guarantees can be made about external data.

Crypto data

This data is used for authentication and encryption.

  • A key derived from your login password is stored on the server (PBKDF2-SHA512). Note that your plain-text password will still pass through the server for processing, but will not be stored.
  • The ciphertext of your secret key is stored on the server, encrypted using either your login password or a dedicated secret key password (AES256-GCM). The secret key will be lost if you lose the password that was used to encrypt it.
  • A random client key is associated with your account and stored on the server.
  • You may elect to store your secret key on your deviceâ€™s local storage, encrypted using the client key (AES256-GCM), to avoid having to enter your secret key password every time.

Encrypted personal data

This data is encrypted using your secret key (AES256-GCM). While this data is stored on the server, it can, realistically, only be accessed using the secret key.

  • Any user data associated with sources and source items, including read state, notes, or other miscellaneous information.

Deletion of data

You can delete your data in the database at any time from the account settings. Note that this action is irreversible.

You can contact the server administrator about deleting data in server logs.
